# 📖 Подробное руководство по сборке и запуску ESPTOOL2 (Для новичков)

Этот файл создан для тех, кто впервые держит в руках платы ESP32 и не знает, с чего начать. Мы пройдём путь от распаковки плат до проведения первой тестовой атаки.

---

## 🛠️ Шаг 1. Подготовка оборудования и понимание ролей плат

Для работы проекта **ESPTOOL2** нам нужны **два** микроконтроллера:

1. **Master (Главный)** — плата **ESP32 DevKit** (или любая обычная ESP32).
   * **Её роль:** Создает Wi-Fi сеть `ESP-Admin`, запускает веб-сервер. Ты подключаешься к ней с телефона или ПК, заходишь на сайт `192.168.4.1` и нажимаешь кнопки. Она передаёт команды на Slave.
2. **Slave (Исполнитель)** — плата **ESP32-S3**.
   * **Её роль:** Принимает команды от Master по беспроводному протоколу ESP-NOW и проводит Wi-Fi/BLE атаки. Она должна быть оснащена антенной (встроенной или внешней) для лучшего сигнала.

---

## 💻 Шаг 2. Установка драйверов на компьютер

Когда ты подключаешь ESP32 к компьютеру по USB-кабелю, компьютер должен распознать её как COM-порт. Для этого нужны драйверы.

1. Переверни плату и посмотри на маленькую прямоугольную микросхему около USB-разъема.
2. **Если там написано CH340 / CH340G / CH340C:**
   * Скачай драйвер с официального сайта производителя: [WCH CH341/CH340 Driver](https://www.wch-ic.com/downloads/CH341SER_EXE.html) (нажми кнопку скачивания).
   * Запусти скачанный файл `.exe` от имени администратора, нажмите **Install**.
3. **Если там написано CP2102 / CP210x:**
   * Скачай драйвер с сайта Silicon Labs: [CP210x USB to UART Bridge VCP Drivers](https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers).
4. Перезагрузи компьютер.
5. **Проверка:** Нажми `Win + X` → выбери **Диспетчер устройств** → разверни вкладку **Порты (COM и LPT)**. Подключи плату к USB — там должен появиться пункт вида `USB-SERIAL CH340 (COMx)` или `Silicon Labs CP210x (COMx)`. Запомни номер порта (например, `COM3`).

## 🔌 Шаг 3. Установка и настройка среды Arduino IDE

Мы будем использовать программу **Arduino IDE** для загрузки прошивки в платы.

### Какую версию выбрать?
* **Рекомендуется:** [Arduino IDE 1.8.19 (Legacy)](https://www.arduino.cc/en/software). Она работает быстрее, не зависает на старых ПК и стабильно прошивает платы.
* Если при попытке скачать с официального сайта тебя блокирует Cloudflare ("Blocked"), скачай её с официального GitHub: [Arduino IDE Releases](https://github.com/arduino/Arduino/releases/tag/1.8.19) (выбери `arduino-1.8.19-windows.exe`).

### Настройка поддержки плат ESP32:
1. Запусти Arduino IDE.
2. Перейди в меню **Файл (File) → Настройки (Preferences)**.
3. В поле **Дополнительные ссылки для Менеджера плат (Additional Boards Manager URLs)** вставь следующую ссылку:
   ```text
   https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
   ```
4. Нажми **ОК**.
5. Перейди в меню **Инструменты (Tools) → Плата (Board) → Менеджер плат (Boards Manager...)**.
6. В поиске введи `esp32`.
7. Найди пакет от **Espressif Systems** и нажми **Установить (Install)**. Подожди окончания загрузки (это займет 2-5 минут).

## ⚡ Шаг 4. Прошивка плат (Инструкция по загрузке)

### 1. Прошивка MASTER (ESP32 DevKit)
1. Открой в Arduino IDE файл прошивки Master: `ESP32_DevKit/WiFi Admin Panel/WiFi_Admin_Panel/WiFi_Admin_Panel.ino`.
2. Подключи плату **Master** (ESP32 DevKit) к ПК через USB.
3. В меню **Инструменты → Плата** выбери: **ESP32 Dev Module** (или NodeMCU-32S).
4. В меню **Инструменты → Порт** выбери COM-порт твоей платы Master.
5. Нажми кнопку **Загрузить** (стрелка вправо в левом верхнем углу).
6. **Важно (Ритуал прошивки):**
   * Смотри на черное окно вывода внизу программы.
   * Как только появится надпись `Connecting...` и побегут точки `......`, **зажми и удерживай кнопку BOOT (или IO0)** на плате Master.
   * Держи её, пока не появится надпись `Writing at...` и не побегут проценты. После этого кнопку можно отпустить.
   * Дождись надписи `Leaving... Hard resetting via RTS pin...` — готово!

### 2. Прошивка SLAVE (ESP32-S3)
1. Открой в Arduino IDE файл прошивки Slave: `ESP32_S3/ESP32_S3_ATTACK/ESP32_S3_ATTACK.ino`.
2. Подключи плату **Slave** (ESP32-S3) к ПК через USB.
3. В меню **Инструменты → Плата** выбери: **ESP32S3 Dev Module**.
4. В меню **Инструменты → Порт** выбери COM-порт твоей платы Slave (он сменится после переподключения кабеля).
5. Нажми кнопку **Загрузить**.
6. Если плата не прошивается автоматически: в момент надписи `Connecting...` зажми кнопку **BOOT**, нажми и отпусти кнопку **RST (EN)**, затем отпусти **BOOT**.

## 📡 Шаг 5. Запуск и использование

1. Отключи платы от компьютера и запитай их (можно подключить к повербанку или блоку питания телефона).
2. Как только обе платы включатся, они автоматически свяжутся друг с другом по протоколу ESP-NOW.
3. Возьми телефон (или ноутбук) и открой поиск Wi-Fi сетей.
4. Подключись к появившейся сети:
   * **SSID (Имя):** `ESP-Admin`
   * **Пароль:** `Esp_div_admin`
5. Открой любой браузер (Chrome, Safari, Firefox) и введи в адресную строку:
   ```text
   http://192.168.4.1
   ```
6. Ты увидишь веб-интерфейс **ESPTOOL2**!
7. Нажмите кнопку **🔄 Проверить** под надписью статуса. Если индикатор загорелся зелёным и написал **«Slave ESP32-S3: Онлайн»**, значит платы успешно нашли друг друга!

## 🎯 Проведение тестов (Wi-Fi и BLE)

* **Beacon Spam (Спам точками):** Перейди на вкладку Wi-Fi, выбери Beacon Spam и нажми **▶️ Запустить**. Вокруг начнут генерироваться фейковые сети `FreeWiFi_XXXX`. Чтобы остановить — нажми **⏹️ Остановить**.
* **Sour Apple (AirDrop спам):** Перейди на вкладку Bluetooth, выбери Sour Apple, нажми **▶️ Запустить**. Все устройства Apple (iPhone, iPad) поблизости начнут получать спам-окна настройки AirDrop.

## ❓ Решение частых проблем (Troubleshooting)

### Ошибка: `Failed to connect to ESP32: Timed out waiting for packet header`
* **Причина:** Плата не перешла в режим прошивки.
* **Решение:** Внимательнее следи за надписью `Connecting...` в Arduino IDE. Зажимай кнопку BOOT на плате ровно в этот момент. Если не помогает, зажми BOOT, нажми EN/RST, отпусти EN/RST, затем отпусти BOOT и запускай прошивку.

### Статус платы Slave показывает «Офлайн» на сайте
1. Убедись, что обе платы включены и находятся в радиусе действия друг друга (до 50-100 метров).
2. Проверь в Мониторе порта (Serial Monitor в Arduino IDE на скорости `115200`), что пишет плата Slave при запуске. Там должна быть строка: `✅ SLAVE готов к приёму команд`.
3. Убедись, что обе платы прошиты именно нашими последними версиями скетчей, где зафиксирован Wi-Fi канал 1.

### Компьютер не видит COM-порт платы
* Попробуй сменить USB-кабель (некоторые кабели поддерживают только зарядку, но не передают данные).
* Попробуй вставить кабель в другой USB-порт компьютера (желательно USB 2.0 напрямую в материнскую плату, если это ПК).
* Проверь установку драйвера CH340 в Диспетчере устройств.
